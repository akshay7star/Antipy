## Environment Variables

Never hardcode secrets (API keys, database URLs) in your code:

```python
import os

# Read from environment
DATABASE_URL = os.environ.get("DATABASE_URL", "sqlite:///local.db")
API_KEY = os.environ.get("API_KEY")
DEBUG = os.environ.get("DEBUG", "False").lower() == "true"
```

Use a `.env` file for local development:
```
DATABASE_URL=postgresql://user:pass@localhost/mydb
API_KEY=sk-12345
DEBUG=True
```

### Practice: Configuration Management

<interactive-code>
import os

class Config:
    """Application configuration from environment variables."""
    
    def __init__(self):
        self.APP_NAME = os.environ.get("APP_NAME", "MyApp")
        self.DEBUG = os.environ.get("DEBUG", "true").lower() == "true"
        self.PORT = int(os.environ.get("PORT", "8000"))
        self.DATABASE_URL = os.environ.get("DATABASE_URL", "sqlite:///app.db")
        self.SECRET_KEY = os.environ.get("SECRET_KEY", "dev-secret-change-me")
    
    def is_production(self):
        return not self.DEBUG
    
    def display(self):
        print(f"App: {self.APP_NAME}")
        print(f"Debug: {self.DEBUG}")
        print(f"Port: {self.PORT}")
        print(f"DB: {self.DATABASE_URL}")
        print(f"Production: {self.is_production()}")
        
        if self.SECRET_KEY == "dev-secret-change-me":
            print("\n⚠️  WARNING: Using default secret key!")
            print("   Set SECRET_KEY env variable in production!")

config = Config()
config.display()
</interactive-code>

## Docker Basics

Docker packages your app and all its dependencies into a **container** — ensuring it runs the same everywhere:

```dockerfile
# Dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "app.py"]
```

## Deployment Checklist

1. ✅ Set `DEBUG = False` in production
2. ✅ Use environment variables for secrets
3. ✅ Set a strong `SECRET_KEY`
4. ✅ Use a production WSGI server (Gunicorn, not Flask's dev server)
5. ✅ Set up HTTPS
6. ✅ Configure logging
7. ✅ Set up monitoring and error tracking

> **Pro Tip:** Use `gunicorn` instead of Flask/Django's built-in server for production: `gunicorn app:app -w 4` runs 4 worker processes for better performance.

> **Common Mistake:** Committing `.env` files to Git. Add `.env` to your `.gitignore` immediately. Leaked API keys are a common security incident.

# FastAPI: The Modern Standard

<ConcurrencyVisualizer />

FastAPI is Python's newest and fastest web framework. It uses **type hints** to automatically validate requests, generate documentation, and provide editor auto-completion. It's 2-3x faster than Flask and Django.

## Basic FastAPI App

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):           # Pydantic model for validation
    name: str
    age: int
    email: str | None = None     # Optional field

@app.get("/")
async def root():
    return {"message": "Hello World"}

@app.get("/users/{user_id}")
async def get_user(user_id: int):    # Type hint = auto validation
    return {"user_id": user_id}

@app.post("/users")
async def create_user(user: User):   # Pydantic = auto body parsing
    return {"created": user.dict()}
```

### Practice: Pydantic Models

<interactive-code>
# Pydantic-style validation (simulated)
class BaseModel:
    def __init__(self, **kwargs):
        for field, field_type in self.__annotations__.items():
            if field in kwargs:
                value = kwargs[field]
                # Basic type checking
                origin = getattr(field_type, '__origin__', None)
                if origin is None and not isinstance(value, field_type):
                    raise ValueError(f"{field} must be {field_type.__name__}")
                setattr(self, field, value)
            elif hasattr(self, field):
                pass  # Has default
            else:
                raise ValueError(f"{field} is required")
    
    def dict(self):
        return {k: getattr(self, k) for k in self.__annotations__}

class Product(BaseModel):
    name: str
    price: float
    in_stock: bool = True
    category: str = "general"

# Valid product
laptop = Product(name="Laptop", price=999.99, category="electronics")
print(f"Created: {laptop.dict()}")

# With defaults
widget = Product(name="Widget", price=4.99)
print(f"With defaults: {widget.dict()}")

# Invalid - missing required field
try:
    bad = Product(price=10.0)  # Missing 'name'
except ValueError as e:
    print(f"\nValidation error: {e}")

# FastAPI would auto-generate Swagger docs at /docs
print("\nAPI docs would be auto-generated at http://localhost:8000/docs")
</interactive-code>

## Key Differences from Flask

```python
# Flask style
@app.route("/items/<int:item_id>", methods=["GET"])
def get_item(item_id):
    return jsonify({"id": item_id})

# FastAPI style — type hints do the work
@app.get("/items/{item_id}")
async def get_item(item_id: int, q: str | None = None):
    return {"id": item_id, "query": q}
```

> **Pro Tip:** Visit `/docs` on any FastAPI app for interactive Swagger UI — you can test endpoints directly in the browser without Postman or curl.

> **Common Mistake:** Forgetting to use `async def` for async operations. FastAPI handles both sync and async, but mixing them incorrectly can hurt performance.

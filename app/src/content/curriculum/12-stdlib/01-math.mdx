# The Math Module

<StdlibVisualizer />

Python's `math` module provides mathematical functions beyond basic arithmetic — trigonometry, logarithms, constants, and more. These are essential for any computation-heavy program.

## Key Constants and Functions

```python
import math

# Constants
math.pi       # 3.141592653589793
math.e        # 2.718281828459045
math.inf      # Positive infinity
math.nan      # Not a Number

# Rounding
math.ceil(4.2)     # 5 — always rounds UP
math.floor(4.8)    # 4 — always rounds DOWN
round(4.5)         # 4 — built-in, rounds to even (banker's rounding!)

# Powers and roots
math.sqrt(16)      # 4.0
math.pow(2, 10)    # 1024.0
math.log(100, 10)  # 2.0 — log base 10
math.log2(1024)    # 10.0
```

### Practice: Math in Action

<interactive-code>
import math

# Distance between two points
def distance(x1, y1, x2, y2):
    return math.sqrt((x2 - x1) ** 2 + (y2 - y1) ** 2)

print(f"Distance (0,0) to (3,4): {distance(0, 0, 3, 4)}")

# Circle calculations
radius = 5
area = math.pi * radius ** 2
circumference = 2 * math.pi * radius
print(f"\nCircle (r={radius}):")
print(f"  Area: {area:.2f}")
print(f"  Circumference: {circumference:.2f}")

# Factorial (used in probability/combos)
print(f"\n5! = {math.factorial(5)}")
print(f"10! = {math.factorial(10):,}")

# Greatest common divisor
print(f"\nGCD(12, 8) = {math.gcd(12, 8)}")
print(f"GCD(100, 75) = {math.gcd(100, 75)}")
</interactive-code>

## Floating Point Precision

Computers store decimals in binary, which causes tiny precision errors:

```python
print(0.1 + 0.2)        # 0.30000000000000004 — NOT exactly 0.3!
print(0.1 + 0.2 == 0.3) # False!
```

**Fix:** Use `math.isclose()` for comparisons or the `decimal` module for exact math:

```python
math.isclose(0.1 + 0.2, 0.3)  # True — allows small tolerance
```

### Practice: Precision

<interactive-code>
import math
from decimal import Decimal

# The classic floating point trap
print(f"0.1 + 0.2 = {0.1 + 0.2}")
print(f"0.1 + 0.2 == 0.3? {0.1 + 0.2 == 0.3}")
print(f"math.isclose? {math.isclose(0.1 + 0.2, 0.3)}")

# For money — use Decimal
price = Decimal("19.99")
tax = Decimal("0.0875")
total = price + (price * tax)
print(f"\nDecimal math: ${total}")

# Never use float for money!
price_f = 19.99
tax_f = 0.0875
total_f = price_f + (price_f * tax_f)
print(f"Float math:   ${total_f}")
</interactive-code>

> **Pro Tip:** For financial calculations, always use `decimal.Decimal` instead of `float`. Floating-point errors compound over millions of transactions.

> **Common Mistake:** Using `round(2.5)` and expecting `3`. Python uses **banker's rounding** — it rounds `.5` to the nearest even number. `round(2.5)` → `2`, `round(3.5)` → `4`.

# Strings: Working with Text

Strings are everywhere in programming — user input, file paths, API responses, database queries, log messages. Python gives you a massive toolkit for manipulating text, and learning these methods will make your code dramatically cleaner.

## Strings Are Immutable

This is the single most important thing to understand: **strings cannot be changed**. Every "modification" creates a **new** string.

```python
name = "Alice"
upper = name.upper()   # Creates a NEW string "ALICE"
print(name)             # Still "Alice" — unchanged
print(upper)            # "ALICE"
```

This means all string methods return new strings — they never modify the original.

## Essential String Methods

<MethodUnit category="string-methods" />

| Method | Purpose | Example | Result |
|--------|---------|---------|--------|
| `.upper()` | ALL CAPS | `"hello".upper()` | `"HELLO"` |
| `.lower()` | all lowercase | `"HELLO".lower()` | `"hello"` |
| `.strip()` | Remove whitespace | `" hi ".strip()` | `"hi"` |
| `.split()` | Split into list | `"a,b,c".split(",")` | `["a","b","c"]` |
| `.join()` | Join list into string | `",".join(["a","b"])` | `"a,b"` |
| `.replace()` | Replace substring | `"hello".replace("l","r")` | `"herro"` |
| `.startswith()` | Check prefix | `"hello".startswith("he")` | `True` |
| `.find()` | Find position | `"hello".find("ll")` | `2` |
| `.count()` | Count occurrences | `"hello".count("l")` | `2` |

## split() and join() — The Power Pair

These two methods are inverses of each other and you'll use them constantly:

```python
# split: string → list
csv_line = "Alice,30,NYC"
parts = csv_line.split(",")    # ["Alice", "30", "NYC"]

# join: list → string
parts = ["Alice", "30", "NYC"]
csv_line = ",".join(parts)     # "Alice,30,NYC"
```

**Key insight:** `.join()` is called on the **separator**, not the list. Think of it as "glue this separator BETWEEN each item."

### Practice: Text Processing

<interactive-code>
# Clean messy user input
raw_input = "   Hello, World!   "
cleaned = raw_input.strip()

# Split a sentence into words

words = sentence.split()

## Checking String Content

Python provides methods to test what a string contains:

```python
"hello123".isalnum()    # True — all alphanumeric
"hello".isalpha()       # True — all letters
"12345".isdigit()       # True — all digits
"  \t\n".isspace()      # True — all whitespace
"Hello World".istitle() # True — title case
```

### Practice: Input Validation

<interactive-code>
# Validate user input
test_inputs = ["Alice123", "42", "hello world", "", "  ", "ADMIN"]

for inp in test_inputs:
    checks = []
    if inp.isalpha(): checks.append("letters only")
    if inp.isdigit(): checks.append("numbers only")
    if inp.isalnum(): checks.append("alphanumeric")
    if inp.isupper(): checks.append("ALL CAPS")
    if not inp.strip(): checks.append("empty/whitespace")
    
    label = ", ".join(checks) if checks else "mixed content"
    print(f"  '{inp}' → {label}")
</interactive-code>

## in Operator and find()

Check if a substring exists or find its position:

```python
text = "Python is great for data science"

# Check existence
"great" in text          # True
"terrible" in text       # False

# Find position (returns -1 if not found)
text.find("great")       # 10
text.find("terrible")    # -1
```

### Practice: Search and Replace

<interactive-code>
# Real-world: clean and process log data
log = "ERROR: Connection timeout at 14:30:05 | WARN: Retry attempt 3 | ERROR: Failed"

# Count errors
error_count = log.count("ERROR")
warn_count = log.count("WARN")
print(f"Errors: {error_count}, Warnings: {warn_count}")

# Extract each entry
entries = log.split(" | ")
for entry in entries:
    level = "ERROR" if entry.startswith("ERROR") else "WARN"
    message = entry.replace("ERROR: ", "").replace("WARN: ", "")
    print(f"  [{level}] {message}")

# Replace
fixed = log.replace("ERROR", "RESOLVED")
print(f"\nFixed: {fixed}")
</interactive-code>

> **Pro Tip:** String methods can be **chained**: `" Hello World ".strip().lower().replace(" ", "_")` → `"hello_world"`. This is clean and common in Python.

> **Common Mistake:** Thinking `.replace()` modifies the original string. It doesn't — strings are immutable. You must **assign** the result: `text = text.replace("old", "new")`.

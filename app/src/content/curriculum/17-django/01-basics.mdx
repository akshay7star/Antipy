# Django: The Full-Stack Framework

Django is Python's most popular web framework. It follows the **"batteries included"** philosophy — authentication, database ORM, admin panel, and more come built-in. Instagram, Pinterest, and Mozilla all use Django.

## The MVT Pattern

Django uses **Model-View-Template** (similar to MVC in other frameworks):

<MermaidDiagram>

  URL["URL Request"] --> VIEW["View: Business logic"]
  VIEW --> MODEL["Model: Database query"]
  VIEW --> TEMPLATE["Template: HTML rendering"]
  MODEL --> VIEW
  TEMPLATE --> RESP["HTML Response"]

| Component | Purpose | File |
|-----------|---------|------|
| **Model** | Database structure | `models.py` |
| **View** | Business logic, request handling | `views.py` |
| **Template** | HTML with dynamic data | `templates/*.html` |
| **URL** | Route mapping | `urls.py` |

## Django Model (ORM)

```python
from django.db import models

class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    published = models.DateTimeField(auto_now_add=True)
    author = models.CharField(max_length=100)
    
    def __str__(self):
        return self.title
```

The ORM lets you interact with databases using Python instead of SQL:

```python
# Create
Article.objects.create(title="Hello", content="World", author="Alice")

# Read
articles = Article.objects.all()
recent = Article.objects.filter(author="Alice").order_by("-published")

# Update
Article.objects.filter(id=1).update(title="New Title")

# Delete
Article.objects.filter(id=1).delete()
```

# Simulating Django's ORM pattern
class QuerySet:
    def __init__(self, data):
        self._data = data
    
    def filter(self, **kwargs):
        result = self._data
        for key, value in kwargs.items():
            result = [item for item in result if item.get(key) == value]
        return QuerySet(result)
    
    def all(self):
        return self._data
    
    def count(self):
        return len(self._data)
    
    def first(self):
        return self._data[0] if self._data else None
    
    def order_by(self, field):
        reverse = field.startswith("-")
        key = field.lstrip("-")
        return QuerySet(sorted(self._data, key=lambda x: x[key], reverse=reverse))

# Simulated database
articles = QuerySet([
    {"id": 1, "title": "Python Basics", "author": "Alice", "views": 1200},
    {"id": 2, "title": "Django Guide", "author": "Bob", "views": 800},
    {"id": 3, "title": "Flask Tutorial", "author": "Alice", "views": 950},
    {"id": 4, "title": "API Design", "author": "Charlie", "views": 2100},
])

# Query like Django
print("All articles:", len(articles.all()))

alice_articles = articles.filter(author="Alice")
print(f"\nAlice's articles: {alice_articles.count()}")
for a in alice_articles.all():
    print(f"  - {a['title']} ({a['views']} views)")

top = articles.order_by("-views").first()
print(f"\nMost viewed: {top['title']} ({top['views']} views)")

## Project Structure

```
myproject/
├── manage.py            # CLI tool
├── myproject/
│   ├── settings.py      # Configuration
│   ├── urls.py          # Root URL routing
│   └── wsgi.py          # Deployment entry
└── blog/                # An "app"
    ├── models.py        # Database models
    ├── views.py         # Request handlers
    ├── urls.py          # App URLs
    ├── admin.py         # Admin panel registration
    └── templates/       # HTML templates
```

> **Pro Tip:** Django's admin panel (`/admin`) gives you a free CRUD interface for your models. Register a model with `admin.site.register(Article)` and you get a full management UI.

> **Common Mistake:** Forgetting to run `python manage.py makemigrations` and `python manage.py migrate` after changing models. Without migrations, your database schema won't update.
